A "schedule resource" is something that can be used for one thing at a time.

Example: A Room (resource) is scheduled for a Meeting (resource use block) titled "Weekly Staff Meeting" tomorrow from 9am to 11am. A mouse click on the block pops up the "owner" and other information about the meeting.

Class SchedResource manages class names, id‘s and labels for a schedule. An instance ties together: 1) A resource class (eg Room) 2) An id that is used to a) select a resource instance and b) select instances of the resource use block class (eg Meeting) The id may be a database id but need not be. It is used only by class methods Resource.find_as_schedule_resource and ResourceUseBlock.get_all_blocks. Not tying this to a database id allows for some extra flexibility and simple optimization. 3) A few strings / html snippets (eg, label, title) for the DOM

Items 1 and 2 are are combined (with a ‘_’) to form a "tag" for the DOM.

See also: ResourceUseBlock (below).
